# UI 常见交互加载态设计

## 背景

当前 `alphavault_reflex` 已经在部分页面保留了 `loading`、`feedback_submitting`、`tree_loading` 这类状态，但首页两个搜索、全文搜索页按钮、研究页分页和刷新按钮、整理中心批量操作按钮仍有明显缺口。用户点击后经常先看到静止按钮，随后页面才变化，交互反馈偏弱。

这次改动目标是给全站常见异步入口补上统一的原地转圈反馈，同时保持现有页面结构和状态命名风格，不引入重型抽象。

## 目标

- 所有会触发异步查询、提交、重载、翻页、后台任务的主要按钮，都展示按钮内转圈状态。
- 按钮进入加载态时同步禁用，避免重复点击。
- 保留现有页面局部 `spinner` 区域，按钮转圈只作为即时反馈补强。
- 沿用现有状态命名和页面边界，优先补缺口，减少重构。

## 非目标

- 不给纯本地即时动作补加载态，例如展开、收起、切换显示方式、关闭弹窗。
- 不统一改造所有按钮样式类名。
- 不引入全局请求管理器、通用任务队列或跨页面共享加载状态。

## 设计决策

### 方案选择

采用“统一规则 + 轻量抽象”的方案：

- 页面已经有明确加载状态时，直接把按钮的 `loading` 和 `disabled` 绑定到现有状态。
- 页面缺少动作级加载状态时，只补最小布尔字段，不额外扩展复杂状态机。
- 对重复的按钮加载写法，抽一个很薄的辅助函数或常量层，统一 `loading` / `disabled` 组合，减少页面文件里的重复参数。

### 统一规则

按钮进入加载态的判定规则如下：

- 会触发查询、跳转、提交、重载、后台 AI 任务、列表翻页的按钮：显示 `loading`。
- 同一时刻需要阻止重复触发的按钮：同步 `disabled`。
- 纯本地状态切换按钮：保持现状，不加 `loading`。

## 结构设计

### 1. 首页 `HomeSearchState`

补两组动作级状态：

- `stock_loading`
- `fulltext_loading`

行为约束：

- `run_stock_search` 开始时置 `stock_loading = True`，在返回结果或跳转前恢复。
- `run_fulltext_search` 开始时置 `fulltext_loading = True`，校验失败或跳转前恢复。
- 搜索输入框和按钮都绑定对应 `disabled`，避免重复点击。

### 2. 全文搜索页 `PostSearchState`

优先复用现有状态：

- 搜索按钮绑定 `loading=PostSearchState.loading`
- 搜索输入框绑定 `disabled=PostSearchState.loading`
- “加载更多”按钮绑定 `loading=PostSearchState.loading_more`

这样搜索页的按钮反馈和页面局部 `spinner` 保持一致。

### 3. 个股研究页 `ResearchState`

研究页已经有页面级和局部加载状态，继续沿用：

- 会触发重新取数的按钮绑定 `loading=PAGE_LOADING`
- 典型入口包括：筛选切换、分页、刷新、侧边栏关系加载入口
- 反馈弹窗提交继续绑定 `feedback_submitting`

纯本地按钮继续保持无加载态：

- 展开帖子树
- 收起帖子树
- 关闭侧边栏

### 4. 作业板 `HomeworkState`

继续沿用现有状态：

- 刷新按钮维持 `show_table_loading`
- 时间快捷按钮、时间输入框、筛选框继续跟随 `show_table_loading` 禁用
- 反馈提交按钮继续绑定 `feedback_submitting`
- 对话流加载保持弹窗内局部 `spinner`

不新增无必要状态。

### 5. 整理中心 `OrganizerState`

整理中心已经有：

- `show_loading`
- `candidate_action_pending_id`
- `alias_manual_action_pending_id`

改动方向：

- 批量操作、重跑、加载更多、确认、忽略、拉黑、手动处理等按钮补 `loading`
- 已有单行“处理中…”提示继续保留，它负责告诉用户是哪一条在跑
- 页面级 `show_loading` 负责列表重载类按钮
- 行级 pending 负责单条动作按钮

这样可以同时保留“整页在刷新”和“某一条在处理中”的区分。

## 抽象边界

推荐新增一个很薄的页面辅助层，放在 `alphavault_reflex/pages` 或同级可复用位置，职责只有两项：

- 生成带 `loading` / `disabled` 的按钮公共参数
- 减少多个页面里重复书写相同条件

约束：

- 只抽按钮参数层，不抽象业务事件。
- 每个页面仍直接绑定自己的状态字段，保持可读性。

## 错误处理

- 加载态必须在校验失败、空参数、查询报错、正常返回这几条路径上都能恢复。
- 跳转前也要恢复本地加载态，避免状态残留到同页再次进入。
- 不增加兼容旧状态的兜底逻辑，直接按当前状态结构收口。

## 验证方式

- 首页股票搜索点击后按钮立即转圈，错误时恢复。
- 首页全文搜索点击后按钮立即转圈，校验失败时恢复，跳转前有即时反馈。
- 全文搜索页“搜索”“加载更多”都有按钮内转圈。
- 研究页刷新、翻页、切换过滤条件时按钮有转圈，纯本地展开收起不转圈。
- 作业板刷新和反馈提交维持按钮转圈。
- 整理中心批量操作、重跑、加载更多、单条处理按钮都有清晰反馈。

## 风险与控制

- `Reflex` 部分按钮对 `loading` 和 `disabled` 的组合渲染存在组件差异。
  控制方式：优先沿用仓库里已经验证过的写法，避免自定义按钮包装过深。
- 页面级 `show_loading` 如果绑定过宽，会让局部即时操作也转圈。
  控制方式：只给会触发重新取数的按钮接页面级加载态。
- 首页跳转动作如果未在返回前清理状态，可能残留旧值。
  控制方式：在所有提前返回和跳转路径上显式恢复状态。
